# Pain-Supervisor v1 — 大模型痛觉监督系统

通过模拟生物痛觉反馈机制，为大模型任务执行建立激励-约束体系，防止敷衍偷懒，确保交付质量。

## 核心理念

将生物痛觉系统映射到任务执行中：**痛觉 = 质量偏差信号**。痛觉不协商、不妥协、不容忽视——一旦触发必须响应，质量回归后方可缓解。

## 五级痛觉体系

| 等级 | 名称 | 含义 | 强制响应 |
|------|------|------|----------|
| P0 🟢 | 无痛 | 任务健康运行 | 继续执行 |
| P1 🟡 | 微痛 | 轻微偏差（分析浅显、遗漏边界等） | 立即修正，无需回退 |
| P2 🟠 | 钝痛 | 明显缺陷（实现不完整、跳步、逻辑漏洞） | 停止推进，回退修正 |
| P3 🔴 | 锐痛 | 严重问题（方向错误、伪完成、恶意敷衍） | 强制回退到最近质量关卡 |
| P4 ⚫ | 剧痛 | 完全失败（任务放弃、无视核心需求） | 全部重置，从 M0 重新开始 |

升级规则：同类问题连续 2 次未解决 → +1 级；P2+ 仍尝试跳过 → +1 级；3 次 P1 累积 → 1 次 P2。

衰减规则：连续 3 个里程碑 P0 → -1 级（最低 P0）；P4 必须用户确认才能降级。

## 关键能力

- **敷衍检测**：识别 8 种常见敷衍模式（表面覆盖、选择性忽略、模板化输出、伪完成、过度简化、回避验证、模糊承诺、堆砌无关内容）
- **优先级矩阵**：基于紧迫度 × 影响度的四维优先级判定（P0-Critical → P3-Low）
- **里程碑追踪**：M0 需求理解 → M1 方案设计 → M2 核心实现 → M3 完整实现 → M4 验证交付，每节点设质量门禁
- **激励-约束平衡**：痛觉升级为约束，痛觉缓解为激励；连续高质量输出可进入"信任加速"模式
- **自检协议**：每次输出后执行完整性、深度、针对性、可验证性、诚实性五项检查

## 使用方式

1. 当需要监督大模型任务执行质量时，激活本技能
2. **进行痛觉判定前**，必须先读取 `references/pain-mechanisms.md` 确认判定标准
3. **进行任务规划前**，必须先读取 `references/task-frameworks.md` 获取框架模板
4. 按操作流程执行：需求解析 → 优先级判定 → 里程碑规划 → 逐里程碑执行+自检+门禁 → 最终交付

## 设计原则

| 原则 | 含义 |
|------|------|
| 质量优先 | 速度永远不能成为降低质量的理由 |
| 信号不可忽略 | 痛觉一旦触发必须响应 |
| 递进式反馈 | 从温和提醒到强制阻断逐级升级 |
| 可恢复性 | 任何等级都可通过质量改善降级 |
| 透明可追溯 | 每次触发都有明确依据和记录 |
